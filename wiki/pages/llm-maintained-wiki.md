# LLM-Maintained Wiki

An LLM-maintained wiki is a small project memory layer that sits between raw
source material and durable documentation.

Use it for experiment results, reflections, recurring mistakes, preferences,
design background, and reusable project context that future agents should not
rediscover from scratch.

## Belongs Here

- Cross-cutting concepts that explain how the project works.
- Lessons from failed attempts or surprising fixes.
- Decision context that is useful but not yet durable documentation.
- Source notes and external entity notes that need careful re-checking.

## Belongs Elsewhere

- Stable public docs belong in durable documentation.
- Open tasks and unresolved decisions belong in issues or the project tracker.
- Secrets, credentials, and sensitive personal data do not belong in the wiki.
